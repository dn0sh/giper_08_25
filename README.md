# Система нейроанализа звонков с функциями поиска, оценки качества и визуализации метрик

## Описание проекта
Разработка прототипа нейросетевого инструмента для анализа разговоров с клиентами с целью автоматизированного контроля качества работы операторов, супервайзеров и отделов продаж. Проект реализует:
- Контекстный семантический поиск
- Многоуровневую проверку качества
- Соответствие требованиям ФЗ-152 по защите персональных данных
- Единый локальный интерфейс для Windows-среды

## Мои задачи и вклад
- Разработка backend-системы на FastAPI с модулями транскрибации, анонимизации и анализа метрик
- Настройка развёртывания через Docker (CPU/GPU), написание скриптов запуска, добавление тестов и документации
- Оптимизация модулей по шумоподавлению, логированию и структуре проекта
- Интеграция LLM-анализа в пайплайн и расширение функциональности по чек-листам, скриптам, эмоциям и типам звонков

## Использованные технологии
- **ASR-модели:** WhisperX large, T-One
- **ML-модели:** Aniemore (WavLM/WavLMBertFusion), E5-large, BGE-M3, GPT-4
- **Backend:** FastAPI, PyTorch, Transformers
- **Поиск:** BM25, векторный поиск (E5/SBERT), гибридное ранжирование
- **Визуализация:** Streamlit, Power BI, Recharts
- **Деплой:** Docker, Nginx/HTTPS
- **Базы данных:** PostgreSQL
- **Фреймворки:** Python, JavaScript, React, Tailwind

## Результаты
- Создан мультикомпонентный пайплайн: транскрибация, анонимизация, скрипты, эмоции, поиск, визуализация
- Достигнута точность транскрибации ~85% WER на выборке из 30+ звонков
- Реализована анонимизация с фильтрацией PII с соблюдением ФЗ-152
- Внедрён гибридный поиск (BM25 + E5) с ручной проверкой релевантности и подсветкой фрагментов
- Запущена LLM-система оценки по скриптам (на базе GPT-4) с сопоставлением результатов с чек-листами
- Разработан Streamlit-интерфейс с UX/UI-макетами для финального слоя
